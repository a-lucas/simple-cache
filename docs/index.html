simple-url-cache

simple-url-cache

Conditionally cache your URL's content on FS or REDIS with RegExp. Also supports cache instance sharing and isolation.

Installation

npm install simple-url-cache

Cache Engine

setters /getters

constructor

constructor( defaultDomain: string, instanceName: string, storageConfig: Object, cacheConfig: Object)

defaultDomain Every URL that miss a hostname will get classified under this domain

instanceName The isolated instance where this cacheEngine will store urls. If another cacheEngine has the same storage type and the same instance name, they will share the pool.

storageConfig File Storage Config or Redis Storage Config Defines how & where url content is stored

cacheConfig Cache config Supports TTL and inclusion/exclusion for any URL rules you need

Example: 


var CacheEngine = require('simple-url-cache');

var engine1 = new CachEngine('http://localhost:3333', 'I1', {type: 'file', dir: '/var/cache'}, cacheRules); 
var engine2 = new CachEngine('http://localhost:4444', 'I1', {type: 'file', dir: '/var/cache'}, cacheRules);
var engine3 = new CachEngine('http://localhost:5555', 'I2', {type: 'file', dir: '/var/cache'}, cacheRules);

// At this stage, engine1 and engine2 share the same pool.

engine1.url('http://a.com/index.html').set('some content'); 
// resolve(true)

engine2.url('http://b.com/index.html').set('some content');
// resolve(true)

engine1.url('http://b.com/index.html').set('some content'); 
// resolves(false) - already cached

engine3.url('http://a.com/index.html').set('some content');
// resolve(true)

engine1.url('http://b.com/index.html').get() 
//resolve(true) -> shared pool with engine1 

engine3.url('http://b.com/index.html').get()
// reject(false) -> not set

The resulting folder structure is: 

# Instance 1
/var/cache/I1/htt[://a.com/index.html
/var/cache/I1/htt[://b.com/index.html

#instance 2
/var/cache/I2/htt[://a.com/index.html

url

url(url: string): CacheStorage

url Initialize a new CacheStorage instance ready to be get(), set(), delete() and has().

Static methods

instanceName & storageType parameters

All these static methods share these same two parameters.

instanceName This is the string defining the instance you initialized your CacheEngine object with. storageType Either file or redis. You can cache for the same instance with these two storage engines simultaneously.

Example: 

var CacheEngine = require('simple-url-cache');

var engine1 = new CachEngine('http://localhost:3333', 'I1', {dir: '/var/cache'}, cacheRules); 
var engine2 = new CachEngine('http://localhost:3333', 'I1', {redisOptions: '....' }, cacheRules);

engine1.url('http:/a.com/test').set('content')
    -> resolves true

engine2.url('http:/a.com/test').set('content')
    -> resolves true

engine1.url('http:/a.com/another_test').set('content')
    -> resolves true

engine2.url('http:/a.com/another_test').get()
    -> rejects

getStoredHostnames

getAllCachedURL(storageName: string, storageType: string): Promise<string[]>

Retrieves an array of all the domains cached.

example: 

var CacheEngine = require('simple-url-cache');

var engine1 = new CachEngine('http://localhost:3333', 'I1', {dir: '/var/cache'}, cacheRules); 

engine1.url('http://a.com/index.html').set('content').then( ... )
engine1.url('http://b.com/index.html').set('content').then( ... )

CacheEngine.getStoredHostnames('I1', 'file').then(function(results) {
    console.log(results);
    // ['http://a.com', 'http://b.com']
});

domain if none provided, then the default domain will be used

getStoredURLs

getCachedDomains(instanceName:string, storageType: StorageType, domain:string): Promise<string[]>

Get the array of cached URLs associated with this domain & instance

domain All the stored URLs retrived had this domain prepended

example:

var CacheEngine = require('simple-url-cache');

var engine1 = new CachEngine('http://localhost:3333', 'I1', {dir: '/var/cache'}, cacheRules); 

engine1.url('http://a.com/index.html').set('content').then( ... )
engine1.url('http://a.com/about.html').set('content').then( ... )

CacheEngine.getStoredURLs('I1', 'file').then(function(results) {
    console.log(results);
    // ['/index.html', '/about.html']
});

geters & setters

delete

delete(): Promise<boolean>

Resolve to true if the url has been suppressed, false if the url wasn't cached Reject an Error if any

get

get(): Promise<string>

Resolve to the url's content Reject if the url wasn't cached

has

has(): Promise<boolean>

Resolve to true if the url is cached, false if the file is not cached, rejected on error

set

set(content: string [, force: boolean]) : Promise<boolean>

Resolve to true if the url has been cached successfully, Rejects false if 

- the url matches the `never` rule.
- The url has already been cached

Rejects on Error

html: the content of the file to be cached, must be UTF8

force: 

- Actualize the TTL for maxAge already cached urls
- Force the caching for url matching the `never` rule.

methods

getCategory()

Returns the url's internal category name. always, maxAge or never

getDomain()

Returns the domain which the URL has been stored with.


var url = CacheEngine.url('http://a.com/index.html');

getInstanceName()

The instanceName sed when this url has been stored

var CacheEngine = require('simple-url-cache');

var engine1 = new CachEngine('http://localhost:3333', 'I1', {dir: '/var/cache'}, cacheRules); 
var engine2 = new CachEngine('http://localhost:3333', 'I2', {dir: '/var/cache'}, cacheRules); 

var url1 = engine1.url('http://a.com/index.html')

url1.getInstanceName() // I1

getStorageType()

Same as getInstanceName(), will return either file or redis

Storage engines

So far, only redis and fs are supported, but it is not hard toadd some more, PR are welcome.

The usage across both storage engines is exactly the same, only the storage config used when initiliazing the CacheEngine differs.

The API behave exactly the same, whenever you use redis or file`

Config Files

Cache Config

This is an object describing which URL will be cached, which URLs won't be cached, and which ones will have a ttl expiration.

This is the same object, independently of the storage engine used.

An example worth 1000 words : 


exports.cacheConfig = {
    // Will cache all URL starting with /posts/ and ending with html for 24 hours
    cacheMaxAge: [ 
        {
            regex: /^\/posts.*html$/,  
            maxAge: 3600
        }
    ],
    // Will cache about-us.html, contact-us.html and /prices.html indefinitively
    cacheAlways: [  
        {
            regex: /^about-us\.html$/, 
            regex: /^contact-us\.html$/,
            regex: /^prices\.html$/
        }
    ],
    // will never cache the url /sitemaps.html
    cacheNever: [ 
        {
            regex: /^sitemaps\.html$/
        }
    ], 
    // If no URL is matched against these rules, then the default is to never cache it. can be 'never' or 'always'
    default: 'never' 
};

File storage config

The simplest config file out there.


export.fileStorageCOnfig = {
    dir: '/var/cache'
}

Redis storage config

A bit more complex. The library noderedis is used here, so a valid redis node config file is needed.

example : 


export.redisStorageConfig = {
    host: '127.0.0.1',
    port: 6379,
    socket_keepalive: true
}
